====================================================================================================
VARIANT B — Multi-scale HKS-filtration vicinity PD/PI for Link Prediction
Spec: docs/superpowers/specs/2026-05-28-diffusion-features-lp-design.md
====================================================================================================

WHAT B IS:
  The existing TLCGNN_LP_FILTER=hks vicinity-PI (|HKS_u-HKS_v| edge filter, ONE
  diffusion scale t) EXTENDED to K scales: compute the HKS-filtration extended-
  persistence image at each t in {0.1, 1.0, 10.0} and STACK (concatenate) the
  per-scale 25-dim PI vectors -> a 75-dim per-edge feature.  Same vicinity-
  geodesic + extended-persistence + PI machinery as exact-PI; only the per-edge
  filter value (Ollivier-Ricci -> multi-scale HKS dissimilarity) changes.
  Guarded behind TLCGNN_LP_FILTER=hks_multi + TLCGNN_HKS_SCALES; default unchanged.
  Computed on the leakage-free training graph (val/test pos removed), like the PI.

  [Cora] B COLLAPSES at test (inherits §14 membership artifact).
      B test_cvAUC=0.500 vs exact test_cvAUC=0.500 (both ~0.5 ⇒ no test signal); B train_cvAUC=0.727.
  [Chameleon] B COLLAPSES at test (inherits §14 membership artifact).

  INTERPRETATION:
  B is a VICINITY-based feature: for a candidate test edge (u,v) the edge is
  removed from the training graph at test, so the local vicinity persistence
  vanishes — at EVERY diffusion scale.  Stacking K scales therefore cannot
  rescue test signal; B collapses (test_pos ≈ test_neg ≈ 0, cvAUC ≈ 0.5) exactly
  like the exact Ollivier-Ricci vicinity-PI.  This CONFIRMS the §14 membership
  artifact is a property of *vicinity construction*, not of the Ricci filtration.
  (Contrast: Variant A, node-level multi-scale HKS, is edge-presence-independent
  and is handled by a separate agent — the predicted dissociation between the two.)
